# Building

Requirements:
- Rust toolchain (install via rustup)

Nice-to-have:
- [`just` task runner](https://just.systems/)
- [`cargo-insta` testing tool](http://insta.rs/docs/quickstart/)

Check the [`justfile`](./justfile) for commands to run common tasks.

# Making releases

Releases go through a pull request. Once the PR is merged, automation pushes
the tag and `cargo-dist` publishes the GitHub Release.

## 1. Open the release PR

For version `X.Y.Z`:

1. Cut a branch: `git checkout -b release/X.Y.Z`
2. Edit [`CHANGELOG.md`](./CHANGELOG.md): rename the `# Unreleased` header to
   `# X.Y.Z` and add a fresh empty `# Unreleased` section above it.
3. Bump `version` in [`Cargo.toml`](./Cargo.toml).
4. Run `cargo build` so `Cargo.lock` picks up the new version.
5. Commit as `release: X.Y.Z` and push.
6. Open a PR with title **exactly** `release: X.Y.Z` and apply the **`release`** label.

## 2. Wait for `release-pr` validation

The [`release-pr`](./.github/workflows/release-pr.yml) workflow runs on every
push to a labeled release PR and checks that:

- the PR title is `release: X.Y.Z`
- `Cargo.toml`'s version equals `X.Y.Z`
- `CHANGELOG.md` has a `# X.Y.Z` section

If any of these fail, fix them in the PR before merging.

## 3. Merge

On merge, `release-pr` pushes a `vX.Y.Z` tag pointing at the merge commit,
which triggers [`release.yml`](./.github/workflows/release.yml) (generated by
[`cargo-dist`](https://github.com/axodotdev/cargo-dist), configured in
[`dist-workspace.toml`](./dist-workspace.toml)) to build binaries and publish
the GitHub Release.
